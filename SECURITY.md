# 安全政策

## 支持范围

维护中的默认分支 `main` 与当前发布版本属于安全修复范围。历史分支和已弃用实验代码不保证修复时限。

## 报告漏洞

请不要在公开 Issue、Pull Request 或讨论区披露可利用细节、真实 Token、私钥、用户对话、生产地址或数据库内容。

请通过仓库所有者 GitHub 账户的私信或其公开资料中列出的私密联系方式报告，并提供：

- 受影响版本或 commit；
- 最小化、脱敏的复现步骤；
- 影响范围与可能的缓解方式；
- 方便后续沟通的联系方式。

收到报告后，维护者会确认接收、评估影响，并在修复或缓解完成后协调披露。请给维护者合理的修复窗口，不要在修复前公开可利用细节。

## 安全设计边界

- 不提交 `.env`、证书、数据库、媒体原文、Agent Token、模型密钥、生产地址、SSH 配置或部署路径。公开安装示例只使用无效占位地址；实际端点仅通过私有配置提供。
- 外部 Agent 只能写入 RawEvent；正式记忆由 Working Agent 的证据治理流程创建。
- Graphiti/Neo4j 只保存内部可重建投影，不保存第二份权威真相。
- 生产部署前运行 `scripts/production_preflight.py`，并保留密钥轮换、备份恢复和数据删除传播的运维记录。
