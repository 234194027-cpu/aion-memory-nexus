title: Working Agent Protocol Compatibility
summary: 消除工作模型业务 JSON 与通用 Runtime 传输封装之间的歧义。
---

- 工作 Agent 可接收白名单 `business_state` 的直接 JSON 输出，不再要求模型额外包裹 `final` 字段。
- 直接业务 JSON 仍由 WorkingCoordinator 进行案件、证据、敏感度和正式记忆治理；不获得工具执行或绕过写入边界的能力。
- 非工作 Agent、未知状态、数组和不符合协议的 JSON 继续失败关闭。
- 在每日 96 次普通工作调用之外保留 32 次交互优先额度，历史导入积压不能占用，确保明确记录、纠正、计划和补证回答仍可实时治理。
