version: 2.5.2
date: 2026-07-22
title: Agent Runtime Recovery
summary: 修复工作 Agent 异步生命周期、队列饥饿、维护兼容与企微长连接假在线，并开放可配置资源预算。
---

- 调度器改为复用 FastAPI 主事件循环，Celery 同步入口使用进程级持久异步循环，避免数据库连接池和 LLM HTTP 客户端跨事件循环复用。
- 工作队列只领取到期事件，并优先处理新对话；外部 Agent/API 历史事件进入最多 8 条的微批，避免历史积压阻塞当前用户消息。
- 工作 Agent 每日模型调用上限由硬编码 24 次调整为可配置的默认 96 次，维护调用默认 8 次，并按 `Asia/Shanghai` 自然日重置。
- 已因旧事件循环缺陷永久失败的 RawEvent 可幂等恢复一次；正常重试、租约与正式记忆治理边界保持不变。
- UserMemoryBrief 兼容历史列表或对象形态的正文，维护任务失败状态可持久记录，不再因单条旧数据导致整轮静默回滚。
- 企微长连接增加帧新鲜度、接收任务与心跳存活判定；心跳超时、普通接收异常和断线均会单通道自愈重连。
