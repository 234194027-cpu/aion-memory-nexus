title: Working Agent Fast Queue Drain
summary: 在双 Agent Runtime 后台增加用户范围内的一键排空任务。
---

- 双 Agent Runtime 页面新增“一键快速处理排队事件”，任务异步执行并展示实时进度、模型调用和失败数量。
- 快速任务只处理点击时已经存在的当前用户队列快照，不会跨用户，也不会被随后新到达的消息拖成长任务。
- 管理员快速任务使用独立的 512 批次上限，跳过普通每日额度与 15 分钟等待；普通事件仍以最多 8 条为一个工作批次。
- 快速任务会重建历史事件中已经过期的批次引用，确保当前仍在队列中的同类事件真正合批，避免逐条重复调用模型。
- Worker 意外重启后，快速任务在 90 秒无心跳时自动转为可重启状态，避免后台按钮长期卡在“运行中”。
- 快速任务仍走统一 Working Agent、案件、证据和正式记忆事务，不放宽来源、敏感度、冲突或写入治理规则。
- 任务状态保存在 `MemoryMaintenanceRun`，重复点击复用正在运行的任务，API 请求无需等待队列处理完成。
