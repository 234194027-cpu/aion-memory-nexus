# 受控真实试运行 Runbook

本 Runbook 只定义受控试运行的准备、验收和证据边界。它不替代生产审批，也不会在没有授权的情况下访问企微、模型供应商、个人数据或生产基础设施。

## 准入条件

1. 在目标环境运行 `python -X utf8 scripts/production_preflight.py`；任何 `FAIL` 都阻断试运行。
   - 若 `ENABLE_SCHEDULER=true`，预检要求 `WEB_CONCURRENCY=1`；多 Web worker 必须先关闭进程内调度器，并将调度职责交给一个指定进程。该规则防止重复提醒、重复扫描和重复 embedding 回填，不替代未来的分布式 leader lock。
2. 确认数据库备份、Redis 持久化/备份、向量索引重建输入、媒体原件存储位置和保留策略。向量索引和 Wiki 派生内容不是唯一权威源，均须可由主库重建。
3. 在迁移前完成带时间戳的 PostgreSQL 备份；执行迁移、健康检查和一个隔离用户的读写检查。回滚使用该次备份和对应迁移 downgrade，不能靠删除迁移文件。
4. 记录负责人、环境、试运行窗口、允许的测试账号和明确禁止写入的敏感类别；日志、截图和报告不得包含原始私人文本或凭据。

## RawEvent 提取恢复

迁移 `014_add_raw_event_processing_lease` 为既有 RawEvent 增加可空的 `processing_started_at`、可回填默认值为零的 `processing_attempts` 和仅记录异常类型的 `processing_error`。提取器以 15 分钟租约原子领取 QUEUED 或过期 PROCESSING 事件；工作案件、证据、治理决策、正式记忆写入和 `COMPLETED` 状态由统一工作 Agent 链路完成。调度扫描会把过期租约放回既有 QUEUED 队列，失败事件仍保持 FAILED，避免无依据的无限重试。

> 当前 V3 口径：旧候选模型已完成迁移收口。回滚时先停止提取 worker、确认没有 PROCESSING 事件；迁移回退不得绕过 RawEvent、Working Case、CommittedMemory 或来源记录。

迁移 `015_add_memory_state_transitions` 追加正式记忆的状态转移审计表。它不回填、覆盖或重写旧记忆；降级仅删除新增审计表。生产迁移前应先导出该表（如果已写入），再执行 `alembic downgrade 014`。

## 建议恢复目标

| 范围 | RPO | RTO | 验证证据 |
|---|---:|---:|---|
| PostgreSQL 权威记忆库 | 最近一次成功备份之后的数据需人工确认 | 由实际恢复演练确定 | 新建临时库恢复、迁移 head、用户隔离读写 |
| Redis/队列 | 可重建的任务状态可能丢失 | 由队列恢复策略确定 | 明确在途 RawEvent 的状态和重试结果 |
| Embedding/向量索引 | 允许重建 | 由数据量实测确定 | 删除索引后从 CommittedMemory 重建并做过滤回归 |
| Wiki/关系派生层 | 允许重建，不覆盖 RawEvent | 由重建作业实测确定 | 来源、版本和删除后重建检查 |
| 媒体原件 | 取决于外部对象/卷备份 | 由存储恢复演练确定 | 校验文件哈希与 RawEvent/MediaArtifact 链接 |

以上数字不是已验证的 SLA。首次演练完成前，RPO/RTO 均为“未知”。

## 真实入口验收

| 场景 | 操作 | 期望证据 |
|---|---|---|
| 企微连接 | 使用已配置的测试 Bot 建连、断连、重连 | 连接状态、一次测试消息、无秘密回显 |
| 写入 | 测试账号发送明确的非敏感文本 | RawEvent 仅追加；正式记忆可追溯到案件、决策和该 RawEvent |
| 提取 | 正常返回、429、超时、无效 JSON、断流各一次（如供应商允许） | 案件/失败状态、重试/降级、任务指标；不得伪造成功 |
| 检索 | 当前事实、历史事实、无证据问题、私有/敏感边界 | 引用真实来源；过期/删除/越权记忆不返回 |
| 纠正与忘记 | 创建更正、撤销和删除测试 fixture | 旧版本不作为当前事实；主库、Wiki、关系、索引和缓存按生命周期闭环清理 |

## “忘记”的边界

在线主库、MemorySource、关系、Wiki 映射/版本、Embedding/向量索引与可清理缓存属于代码可验证闭环。备份、运行日志、外部模型请求历史和第三方平台保留期受基础设施/供应商政策约束，必须在试运行记录中单独标注，不能承诺即时物理抹除。

## 试运行结束条件

- 对上述每个已执行场景记录时间、环境、匿名测试标识、结果和残余风险。
- 失败场景必须区分“预期安全降级”“已修复”与“待处理”，不得因为无法复现而标记通过。
- 任何真实数据质量/成本/延迟结论都应由 `docs/memory-eval.md` 的确定性基线之外的、经同意的标注集与运行指标支持。
